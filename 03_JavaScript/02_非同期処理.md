# JavaScript の非同期処理

JavaScript は基本的にシングルスレッドで動作する。これは、一度に一つの処理しか実行できないことを意味する。

そのため、時間がかかる処理（例：ネットワーク通信、ファイル読み込み、setTimeout）があると、その間他のすべての処理（例：UI の更新）がブロックされてしまい、フリーズしたように見えてしまう。

非同期処理は、このような時間のかかる処理をメインの処理の流れから切り離し、「処理が終わったら教えてね」という形で別の場所で並行して実行することで、他の処理をブロックしないようにする仕組みのこと。

## 非同期処理の主な実現方法

| 手法               | 説明                                                                                                                                                                      |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Promise (プロミス) | 非同期処理の最終的な結果（成功または失敗）を表すオブジェクトです。非同期処理の状態（実行中、成功、失敗）を管理し、処理の成功/失敗に応じてコールバック関数を登録できます。 |
| async/await        | Promise をベースにした構文上の糖衣（シンタックスシュガー）です。非同期処理を、あたかも同期処理のように（上から順番に）見やすく記述できます。                              |
| コールバック関数   | 古典的な手法。非同期処理が終わった後に実行される関数を引数として渡す方法ですが、処理が複雑になると「コールバック地獄」と呼ばれる可読性の低いコードになりがちです。        |
