# JavaScript のモジュールの文法

モジュールとは、プログラムを機能ごとに分割し、ファイルとして独立させたもの。

これにより、コードの再利用性、保守性、可読性が向上し、他のファイルとの 名前の衝突（競合）を防ぐことができる。

モジュールシステムには歴史的な経緯がありますが、現代の JavaScript（ES Modules/ESM）では主に以下の構文が使われる。

## エクスポート（書き出し）

他のファイルで利用したい変数、関数、クラスなどを外部に公開します。

- 名前付きエクスポート (Named Exports): 複数の項目を名前付きでエクスポートします。

```JavaScript
// module.js
export const PI = 3.14;
export function sum(a, b) {
  return a + b;
}
```

- デフォルトエクスポート (Default Export): ファイル内で一つだけ、そのモジュールの「主役」となるものをエクスポートします。

```JavaScript
// logger.js
export default class Logger {
  // ... クラスの定義
}
```

## インポート（読み込み）

外部ファイルでエクスポートされた項目を、自分のファイルで利用可能にします。

- 名前付きインポート: 波括弧 {} で必要な項目を正確に指定します。

```JavaScript
// main.js
import { PI, sum } from './module.js';
console.log(sum(1, 2)); // 3
```

- デフォルトインポート: 任意の名前で受け取ります。波括弧は不要です。

```JavaScript
// main.js
import MyLogger from './logger.js';
const logger = new MyLogger();
```

これらの知識が、JavaScript を理解する上での土台となります。特に非同期処理とモジュールは、大規模なアプリケーション開発において非常に重要です。
