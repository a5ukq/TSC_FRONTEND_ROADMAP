# XSS（クロスサイトスクリプティング）

XSS は、サイトを閲覧しているユーザーのブラウザで、攻撃者が用意した不正な JavaScript を実行させる攻撃である。

### 攻撃の仕組み

1. 攻撃者が、掲示板や検索フォームなどの「ユーザーの入力を表示する場所」に、悪意のあるスクリプトを仕込む。
2. 一般ユーザーがそのページにアクセスすると、ブラウザがそのスクリプトを「サイトの一部」と認識して実行してしまう。

### 主な被害

- cookie の摂取：ログイン状態を維持するための情報を盗まれ、アカウントを乗っ取られる。
- 画面の改ざん：偽のログインフォームを表示してパスワードを入力させる。
- ウイルス配布サイトへのリダイレクト：勝手に別のサイトへ飛ばされる。

### 対策

- エスケープ処理（サニタイズ）：`<`を`&1t;`に変換するなど、特別な意味を持つ文字を無害化する。
- CSP（Content Security Policy）の設定：ブラウザに対し、許可したドメイン以外のスクリプト実行を禁止する。

# CSRF（クロスサイトリクエストフォージェリ）

CSRF は、「ログイン中のユーザー」に、本人の意図しない「重要な操作」を強制的に実行させる攻撃である。

### 攻撃の仕組み

1. ユーザーが銀行やSNSなどのサイトにログインした状態（Cookie有効状態）で、攻撃者が作った罠サイト（偽の広告やメールのリンク）を踏む。
2. 罠サイトに仕込まれたコードが、ユーザのブラウザを介して、ログイン中のサイトへ「パスワード変更」や「送金」のリクエストを勝手に送信する。
3. サーバー側は「正規にログインしているユーザからの指示」と思い込み、処理を実行してしまう。

### 主な被害

- 勝手な投稿や購入：SNSでの書き込みや、ECサイトでの注文
- 設定の変更：メールの転送設定やパスワードの書き換え。
- 退会処理：アカウントの強制削除など。

### 対策

- ワンタイムトークンの発行：フォームの送信時に、サーバー側で発行した推測困難な文字列（トークン）を照合する。
- SameSite属性の設定：クッキーに`SameSite＝Lax`または`Strict`を設定し、他使徒からのリクエストにクッキーを載せないようにする。

### XSSとCSRFの比較

| 比較項目     | XSS                                    | CSRF                                     |
| ------------ | -------------------------------------- | ---------------------------------------- |
| 主な狙い     | ユーザーの情報（クッキー等）を盗む     | ユーザーの権限で操作を行わせる           |
| 実行場所     | 被害者のブラウザ上（スクリプト実行）   | 被害者のブラウザ経由（リクエスト送信）   |
| サイトの要件 | サイトにスクリプトを注入できる隙がある | ログイン済みでセッションが維持されている |
| 主な対策     | 出力時のエスケープ処理                 | トークンによるリクエスト検証             |
